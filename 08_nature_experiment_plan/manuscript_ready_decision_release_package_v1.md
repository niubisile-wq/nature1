# Manuscript-Ready Decision and Release Package v1
Date: 2026-07-11

## Purpose
This note turns the current frozen evidence into a manuscript-ready package that mirrors the latest Nature patterns: decision-facing exposure, infrastructure translation, transfer status, and honest release state.

## Comparator Pattern Summary

Nature-family papers in this area now tend to combine:
- high-resolution deformation or subsidence rates,
- population and built-up exposure,
- infrastructure or building risk,
- explicit transfer or control status,
- and release/metadata transparency.

## What Each Piece Does

### Decision-facing exposure matrix
- Summarizes population, built-up, infrastructure, and transfer status in one table.
- Best suited for a main Results / Extended Data table.

### Infrastructure translation
- Freezes the lead-case roads/rail layer.
- Keeps other regions honestly labeled as not yet translated.

### Full all-region hierarchical closure
- Compiles the frozen comparison families into one all-region closure note.
- Keeps the limitation explicit: closure-by-compilation is not the same as a new all-region fit.

### Technical validation
- Consolidates observability, sign/unit, blocked/equal-area, stratified-control, and transfer evidence.

### Release-gap memo
- States clearly that the package is DOI-ready but not DOI-minted.
- Prevents overclaiming in the manuscript release section.

### Release maturity matrix
- Maps Nature / Scientific Data release expectations to local evidence.
- Makes the remaining external gap explicit without weakening the local package.

### Socioeconomic translation gap
- Marks the next possible Nature-style upgrade: a socioeconomic vulnerability companion layer.
- Keeps the next step explicit without overstating current coverage.

### Socioeconomic layer candidate
- Names GRDI as the preferred raster candidate, with SHDI and nightlights as fallback paths.
- Turns the gap into a concrete shortlist instead of a vague future direction.

### Socioeconomic layer trial plan
- Defines the actual GRDI/SHDI matching test before any manuscript integration.
- Keeps the socioeconomic upgrade concrete, testable, and reversible if the harmonization fails.

### Socioeconomic layer access matrix
- Clarifies that SHDI is the immediately executable fallback while GRDI is the preferred but access-gated upgrade.
- Prevents the manuscript from implying that the raster layer is already available when it still needs access work.

### Socioeconomic trial harness
- Provides an executable scaffold that can ingest SHDI or GRDI once the file is staged.
- Keeps the socioeconomic upgrade operational rather than purely documentary.

### Socioeconomic gradient comparison
- Compares frozen GVI against SHDI using national values and within-country subnational spread.
- Makes the internal vulnerability gradient visible for reviewer-facing discussion.

### Structural vulnerability upgrade matrix
- Separates structural vulnerability and infrastructure-access inequality from the current socioeconomic context layer.
- Keeps GVI / SHDI honest as context rather than final closure.

### Comparator scout v4
- Captures the newest literature shift toward structural vulnerability, infrastructure access inequality, and multi-axis burden.
- Ensures the manuscript package does not freeze the wrong vulnerability abstraction.

### Structural vulnerability context gap
- States explicitly that GVI / SHDI are context layers, not a closed structural-vulnerability layer.
- Keeps infrastructure-access inequality separate from generic socioeconomic context.

### Structural vulnerability candidate shortlist
- Gives the next upgrade step concrete candidate directions.
- Keeps the structural-vulnerability gap actionable without pretending it is already closed.

### Structural vulnerability trial plan
- Prespecifies the candidate order and the trial logic.
- Prevents the next upgrade from becoming ad hoc.

### Structural vulnerability access matrix
- Separates globally harmonizable candidates from method-only comparators.
- Prevents U.S.-only indices from being mistaken for a multi-region companion layer.

### Structural vulnerability trial report
- Records the first probe result for the global critical-infrastructure dataset.
- Makes the current limitation explicit: reachable archive, but no usable region overlay yet without a geospatial parser.
- Preserves the candidate as reachable-but-not-yet-usable rather than promoting it prematurely.

### Structural vulnerability proxy summary
- Extracts finite numeric raster summaries from the global critical-infrastructure dataset.
- Moves the source from archive-readiness to numeric proxy feasibility.

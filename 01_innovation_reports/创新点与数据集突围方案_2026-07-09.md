# 创新点与数据集突围方案
日期：2026-07-09

## 一、现在应该怎么办

不要继续把主张写成“更多区域的沉降暴露被低估”。这条路已经被 Nature、Science、Nature Cities、Nature Communications 的近年工作压得太窄。

下一步应该改成构建一个新的数据产品和机制框架：

**Open InSAR Observability Bias and Exposure Benchmark**

中文可写为：

**开放 InSAR 可观测性删失与暴露低估基准数据集**

这条路的核心不是再做一张沉降图，而是回答：

> 哪些最需要沉降监测的地方，反而在公开 InSAR 产品中最容易不可见？这种不可见会把人口、建筑、交通和农业灌溉暴露低估多少？

这个问题比“哪里在沉降”更安全，也更可能避开已有 CNS 撞车。

## 二、可升级成 Nature 级的创新点

### 创新点 1：开放 InSAR 观测债务

定义一个新指标：

`open_insar_monitoring_debt = monitoring_need - open_insar_observability`

其中：
- `monitoring_need` 来自沉降强度、地下水/灌溉压力、干旱压力、人口/建成区/道路暴露。
- `open_insar_observability` 来自 LiCSAR/EGMS/DWR/Japan LiCSBAS 等公开产品的可下载性、时间密度、coherence、valid-pixel mask 和稳定时序可用性。

可写主张：

> Public land-subsidence knowledge has monitoring debt: the places with high need for measurement are not necessarily the places with usable open InSAR evidence.

### 创新点 2：可观测性删失导致风险估计偏差

定义：

`risk_underestimation_factor = exposure_in_strong_motion_and_unobservable / exposure_in_strong_motion`

暴露层不只用人口，还要至少包括：
- GHSL population。
- GHSL built-up surface。
- OSM roads/railways。
- Microsoft/Google building footprints 或 Google Open Buildings Temporal。
- WorldCover cropland / water / wetland。
- FAO GMIA irrigation。

这能把结果从“像元不可见”升级到“公共风险估计被低估”。

### 创新点 3：产品偏差转移函数

当前 WorldCover 模型已经说明：土地覆盖加入后，Chao Phraya、Indus、Rhone 仍稳定，Rhine 消失，Po 转正。这说明低可见性不是单一的强沉降效应，而是产品、地表和区域共同作用。

可写成：

`observability_failure = f(product, landcover, exposure, region, time_sampling)`

Nature 级价值在于：这是一个可迁移的 bias framework，而不是某个地方的沉降案例。

### 创新点 4：基准阶梯，而不是单一真值

把证据分成四层：

| Tier | 数据 | 作用 |
|---|---|---|
| T1 高密度独立基准 | EGMS、DWR/TRE、地方 InSAR | 像元/点级 benchmark |
| T2 稀疏独立锚点 | NGL MIDAS GNSS、GEONET/GSI GNSS | 区域垂向速度 sanity check |
| T3 公开派生变形场 | Nature 2026 delta VLM、Japan LiCSBAS Zenodo | 跨区域外部变形参照 |
| T4 公开观测产品 | LiCSAR coherence/unwrapped/coverage | 被审计对象 |

这能避免“拿 LiCSAR 审计 LiCSAR”的逻辑弱点。

## 三、今天新侦察到的数据突破口

### 1. EGMS API：Nature 正刊最快补强路径，但需要免费 token

官方 Copernicus EGMS 页面确认：
- EGMS Basic 覆盖 EU27、Iceland、Norway、UK。
- 空间分辨率为 5 x 20 m。
- 时间范围 2016-present。
- 2019-2023 数据已验证。
- 官方提供 API notebook 下载路线。

官方 GitHub `copernicus-land/egms-api` 说明：
- 必须取得 CLMS API token。
- 查询 endpoint 是 `https://egms.land.copernicus.eu/insar-api/archive`。
- 支持 `levels`、`releases`、`bbox`、`tileId`、`productType`、`direction` 等筛选。
- 下载链接由 `/download/{filename}?id={query_id}` 构造。

判定：

**如果目标仍是 Nature 正刊，应该放宽“严格无凭据”限制，使用免费 CLMS token。**

严格无 token 路线下，EGMS 只能作为网页/样例支撑，不能形成 Po/Rhone/Rhine/Netherlands 高密度 benchmark。

### 2. Source Cooperative EGMS：只适合样例，不够救 Nature

Source Cooperative 页面显示公开可见，但当前目录层级实际只有 `L2a/parquet/CY`、`cy.parquet`、`cy-pid-geom.parquet` 和 `db.duckdb`。

本机已验证 Cyprus Parquet 样例可下载和解析，但它不能完成：
- Po/Venice benchmark。
- Netherlands benchmark。
- Rhone/Rhine benchmark。

判定：

Source Cooperative 可作为“云原生 EGMS 可读性证明”，不能作为 Nature G5 的主基准。

### 4. Japan LiCSBAS Zenodo：强公开数据集扩展，但不是独立真值

新增元数据：

`C:\Users\刘子轩\radar_outputs\rescue_dataset_probe\zenodo.org_api_records_4243151`

记录信息：

- files: 195
- total size: 28.75 GB
- unique city tokens: 73
- 数据包含 191 个 deformation datasets，以及 LOS velocity、annual deformation、decomposed vertical/EW velocity、rice paddy、NDVI、optical images、topography、SB network。

判定：

这是非常适合扩展“公开 InSAR 数据产品审计”的非欧美数据集，但它源自 LiCSAR/LiCSBAS，不能作为完全独立真值。

### 5. GSI GEONET：日本独立 GNSS 源，但需谨慎下载

GSI 官网说明：
- GNSS CORS observational data 可免费通过互联网下载。
- RINEX data from April 1, 2010 and daily coordinates from 1996 online。
- 同时明确禁止为了收集数据而进行大量下载或 crawler。

判定：

正式论文中可作为日本 GNSS 独立源，但当前自动化应优先使用 NGL 已处理的 GEONET/MIDAS 速度，避免大规模爬取 GSI。

## 四、数据集应该怎么搭

### 数据产品名称

建议英文名：

**Open InSAR Observability Bias and Exposure Benchmark**

建议中文名：

**开放 InSAR 可观测性删失与暴露低估基准数据集**

### 数据表结构

1. `region_catalog`
   - region_id
   - region_type: delta / city / valley / coastline
   - bbox
   - data_tier
   - primary_benchmark_source
   - exposure_sources

2. `open_insar_observability`
   - region_id
   - product: LiCSAR / EGMS / DWR / Japan_LiCSBAS
   - pair_or_epoch
   - coherence_threshold
   - any_observable
   - annual_observable
   - majority_observable
   - stable_time_series_observable

3. `benchmark_motion`
   - region_id
   - source: DWR / EGMS / NGL_GNSS / Japan_LiCSBAS / Nature_delta_VLM
   - lon
   - lat
   - velocity_mm_yr
   - uncertainty
   - benchmark_tier

4. `exposure_overlay`
   - region_id
   - exposure_type: population / built_up / road / railway / building / cropland / irrigation
   - source
   - total_exposure
   - strong_motion_exposure
   - unobservable_strong_motion_exposure

5. `bias_metrics`
   - region_id
   - threshold
   - product
   - risk_underestimation_factor
   - spatial_block_bootstrap_ci
   - dominant_landcover_group
   - notes

## 五、三条可执行路线

### 路线 A：Nature 正刊突围路线

要求：
- 使用 CLMS 免费 token 下载 EGMS。
- 完成 Po/Rhone/Rhine/Netherlands 至少一个高密度欧洲 benchmark。
- 加入 NGL GNSS 做独立锚点。
- 用 GHSL/OSM/buildings 做面积加权暴露低估。

优点：
- 最可能把 G5 从 fail 拉到 pass。
- 审稿人最难否认 EGMS 的基准质量。

缺点：
- 不再是严格无凭据路线。
- 需要 token 和较大下载空间。

当前判定：

**这是冲 Nature 正刊唯一现实路线。**

### 路线 B：严格无凭据数据产品路线

数据源：
- LiCSAR/CEDA。
- DWR/TRE Central Valley。
- NGL MIDAS GNSS。
- Japan LiCSBAS Zenodo。
- Nature 2026 delta VLM。
- GHSL/WorldCover/OSM/GMIA。

优点：
- 全部可脚本化公开访问。
- 可以形成一篇很强的 Nature Communications / Scientific Data / Earth System Science Data 级数据论文。

缺点：
- EGMS 高密度欧洲 benchmark 缺失。
- Nature 正刊仍偏弱。

当前判定：

**如果不能使用 CLMS token，这条路线更诚实。**

### 路线 C：非欧美高动态案例路线

候选：
- Mexico City：NGL GNSS 有强沉降锚点，文献和 NASA/NISAR 热度高。
- Chao Phraya：本地模型稳定正向，GNSS 有 6 个点。
- Japan Kanto/Niigata：GNSS 很密，Japan LiCSBAS 数据完整。

优点：
- 更容易形成“高风险区观察债务”的故事。
- 能避免欧洲/美国过度饱和。

缺点：
- Mexico/NISAR/ARIA/OPERA 仍可能遇到 Earthdata 或作者数据问题。
- Japan 数据是 LiCSAR 派生，独立性不足。

当前判定：

**适合作为路线 A/B 的第二图组，不适合作为唯一主线。**

## 六、我建议的下一步执行顺序

1. 准备 EGMS API downloader scaffold
   - 不把 token 写入代码。
   - 自动寻找本地 `token.jwt`。
   - 若 token 不存在，只输出 Po/Rhone/Rhine/Netherlands 的 query payload。

2. 先做 NGL-GNSS 锚点图
   - 把 NGL 站点叠到 Po、Rhone、Rhine、Chao Phraya、Mexico City。
   - 检查 GNSS up velocity 与 Nature VLM / DWR / LiCSAR failure 的方向是否矛盾。

3. 把 exposure overlay 升级为面积加权
   - GHSL population and built-up 先做。
   - OSM roads/railways 做 network exposure。
   - 选 1-2 个区域补 Microsoft/Google building footprints。

4. 做一个最小可发表数据集版本
   - Central Valley：DWR + LiCSAR + NGL + GHSL/OSM。
   - Po/Rhone/Rhine：EGMS 或 NGL + LiCSAR + GHSL/OSM。
   - Chao Phraya：Nature delta VLM + NGL + LiCSAR + WorldCover/GHSL。
   - Japan Kanto/Niigata：Japan LiCSBAS + NGL + WorldCover/GHSL。

5. 再判断能否重回 Nature 正刊
   - EGMS 成功：继续冲 Nature。
   - EGMS 失败：转 Nature Communications / Scientific Data，更稳。

## 七、当前最关键判断

**只靠继续找更多 LiCSAR delta，不会把项目救成 Nature。**

真正能救的是：

1. EGMS 高密度 benchmark。
2. NGL/GNSS 独立锚点。
3. 面积加权 exposure underestimation factor。
4. 一个可发布的数据产品，而不是几个案例图。

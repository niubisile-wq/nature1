现在不应该继续无差别增加 delta 或城市样本。要把 Nature 正刊路线救回来，最关键的是补上独立高质量 benchmark：

## 下一步命令
如果拿到 CLMS `token.jwt`：

```powershell
py prepare_egms_rescue_queries.py --token-jwt C:\path\to\token.jwt
```

如果已有短期 bearer token：

```powershell
py prepare_egms_rescue_queries.py --access-token $env:EGMS_ACCESS_TOKEN
```

脚本会生成：
- `egms_search_results.json`
- `egms_hits.csv`

其中 `egms_hits.csv` 会包含每个 EGMS 文件的 `download_url`，可作为后续下载 manifest。

## Nature 数据集设计
EGMS 原始数据是复用的 Copernicus/CLMS 数据，不能包装成我们原创数据。真正可发表的数据集应是派生 benchmark：

**Open InSAR Observability Bias and Exposure Benchmark**

建议组成：
- AOI 网格和统一投影
- 每个产品的 valid-pixel / coherence / time-density / downloadability mask
- EGMS benchmark deformation layer
- LiCSAR/公开产品 observability layer
- WorldCover/GHSL/OSM/building footprint exposure layer
- `open_insar_monitoring_debt`
- `risk_underestimation_factor`
- 模型输入表、bootstrap 结果、figure source data
- README、数据字典、脚本版本、第三方数据版本和许可证映射

仓储策略：
- 派生 benchmark 小到中等规模：Zenodo 或 Dryad 可行
- 空间环境大数据、长期版本化：PANGAEA 或机构仓储更合适
- 代码：GitHub release + Zenodo DOI
- 第三方大数据：不重复分发，给出原始 DOI/URL、版本和处理脚本

## 当前 Go/No-Go
**Nature 正刊：Conditional GO。**

条件只有一个：拿到 EGMS/CLMS token 后，Po、Netherlands、Rhone 至少两个 AOI 能证明：
1. EGMS 中存在可用高密度形变参照；
2. LiCSAR/开放产品的可观测性缺口不是随机缺失；
3. 缺口会系统性改变人口、建成区、道路或农业灌溉暴露估计；
4. 这个偏差能被 land-cover/product/region/time-sampling transfer function 解释。

如果 EGMS 不能闭环，主线应降级为：
**Nature Communications / Scientific Data / ESSD 数据集论文路线。**
